The problem

AI agents are fast. But they make structural mistakes.

AI coding agents can scaffold an entire feature across a dozen files in seconds. But they make structural mistakes that are easy to miss during review:

  • An import references a module that was never created
  • A function signature changed but not all callers were updated
  • A file was renamed but old references remain in three other files
  • TODO placeholders were left behind as "implementation notes"
Features

Everything you need to validate AI-generated code

Static analysis that works offline

Seven analyzers across 5 languages run locally against your git diff. No API calls, no network access, no data leaves your machine. Works on planes, in air-gapped environments, and in CI.

Multi-language support

Parses and validates Go, TypeScript, Python, Java, and Rust. Understands imports, exports, function signatures, and cross-file references for each language.

Git native

Installs as a pre-commit hook with fault hook install. Reads staged, unstaged, or branch diffs directly from git. Outputs terminal, JSON, or SARIF.

Auto-fix engine

Run fault fix to automatically repair broken imports, replace hardcoded secrets with env vars, and remove debug statements. Preview changes with --dry-run.

Watch mode

Run fault watch for continuous validation as you edit. Detects file changes in real-time and re-runs analysis within milliseconds — the tightest feedback loop for AI coding.

GitHub PR review

Post inline review comments directly on pull requests via GitHub Actions. Findings appear right in the diff where developers already review code.

Security & hallucination detection

Catches hardcoded secrets, insecure crypto, references to non-existent APIs, phantom packages, and invented function signatures that AI agents hallucinate.

VS Code extension

See Fault findings inline as you code. The VS Code extension runs analysis on save and shows diagnostics, quick-fixes, and code actions right in the editor.

Pro

AI-enhanced validation

LLM-powered confidence scoring rates how likely each changed file is to be correct. Optional spec comparison flags when changes deviate from your requirements.

How it works

Three steps to validated commits

1

AI agent makes changes

Claude Code, aider, or Cursor edits files across your project — new endpoints, refactored modules, updated types.

2

Fault validates the diff

When you commit, fault check --staged runs automatically via the pre-commit hook. It parses every changed file, resolves imports and references, and runs seven analyzers in parallel.

3

Auto-fix or commit with confidence

If Fault finds errors, the commit is blocked. Run fault fix to auto-repair what it can, or fix manually. If everything passes, the commit proceeds. No false confidence, no missed breakage.

Get started in 30 seconds

# Install
curl -sSf https://fault.jabbey.io/install.sh | sh

# Set up your project
cd your-project
fault init
fault hook install

# That's it. Next time you commit, Fault validates your changes.

# Or use interactively:
fault watch # continuous validation as you edit
fault fix # auto-fix broken imports, secrets, debug logs
