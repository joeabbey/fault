The problem

AI agents are fast. But they make structural mistakes.

AI coding agents can scaffold entire features across a dozen files in seconds. But they make mistakes that slip through code review:

  • An import references a package that doesn't exist in your manifest
  • A function signature changed but not all callers were updated
  • Error handling was silently swallowed — _ = err in Go, empty catch {} in Java
  • Exported code is now dead — nothing imports it anymore
  • Circular dependencies were introduced between modules
  • TODO placeholders and debug statements were left behind

Features

Everything you need to validate AI-generated code

Works completely offline

All ten analyzers run locally against your git diff. No API calls, no network access, no data leaves your machine. Works on planes, in air-gapped environments, and in CI.

Manifest-aware validation

Validates imports against your actual project dependencies — package.json, go.mod, Cargo.toml, pom.xml, build.gradle, .csproj, composer.json, Gemfile, and Package.swift.

Git native

Installs as a pre-commit hook with fault hook install. Reads staged, unstaged, or branch diffs directly from git. Outputs terminal, JSON, or SARIF.

Auto-fix engine

Run fault fix to automatically repair broken imports, replace hardcoded secrets with env vars, and remove debug statements. Preview changes with --dry-run.

Watch mode

Run fault watch for continuous validation as you edit. Detects file changes in real-time and re-runs analysis within milliseconds.

GitHub PR review

Post inline review comments directly on pull requests via GitHub Actions. Findings appear right in the diff where developers already review code.

New

Error handling detection

Catches discarded Go errors (_ = err), unhandled TS promises, bare Python except:, empty Java catch blocks, and Rust .unwrap() outside tests.

New

Dead code & dep graph

Detects exported symbols that nothing imports, circular dependencies between modules, and unused entries in your manifest files.

VS Code extension

See Fault findings inline as you code. The VS Code extension runs analysis on save and shows diagnostics, quick-fixes, and code actions right in the editor.

Pro

AI-enhanced validation

LLM-powered confidence scoring rates how likely each changed file is to be correct. Optional spec comparison flags when changes deviate from your requirements.

How it works

Three steps to validated commits

1

AI agent makes changes

Claude Code, aider, or Cursor edits files across your project — new endpoints, refactored modules, updated types.

2

Fault validates the diff

When you commit, fault check --staged runs automatically via the pre-commit hook. It parses every changed file, resolves imports and references, and runs ten analyzers in parallel.

3

Auto-fix or commit with confidence

If Fault finds errors, the commit is blocked. Run fault fix to auto-repair what it can, or fix manually. If everything passes, the commit proceeds.
